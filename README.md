## 🧪 Git運用での学習（重要）

本プロジェクトでは、Git運用の改善も同時に実施した。

### 発生した問題
- detached HEAD状態でのコミット発生
- push時にブランチ不整合エラー
- バックアップスクリプトによる無限ネスト問題
- `.next`フォルダ混入によるパス長エラー

### 原因
- ブランチ状態をチェックせずにcommitを実行
- backup処理でプロジェクト全体を再帰コピー
- ビルド成果物（.next）をGit管理対象に含めていた

### 改善内容
- `.gitignore`による不要フォルダ除外
- backup対象の明確化（node_modules / .next / backup除外）
- Git操作前のブランチチェック追加
- ローカル優先同期スクリプト導入

---

## 🛡 Git運用ルール（現在）

- 作業前に必ず状態確認

## 🧠 学習ポイント

- Gitは「保存ツール」ではなく「履歴管理システム」
- detached HEADはブランチ外状態での作業
- backup設計を誤ると指数的にデータが増殖する
- .gitignore設計がプロジェクト安定性の鍵
