【現在の構成】

■ backend/
- db.php
  ・SQLite接続関数 getDB() を提供
  ・formsテーブル（フォーム定義）
  ・answersテーブル（回答保存用）

- save.php
  ・フォームの新規作成 / 更新API
  ・POST JSON受信
  ・fieldsはJSON文字列でDB保存
  ・formsテーブルへUPSERT的処理（id有無で分岐）

- list.php
  ・forms一覧取得API
  ・fieldsは返さない（軽量化）
  ・SELECT id, title, description FROM forms ORDER BY rowid DESC

- detail.php
  ・フォーム詳細取得API
  ・GET idで取得
  ・fieldsはJSON decodeして配列に変換して返す
  ・レスポンス:
    {
      status: "success",
      id,
      title,
      description,
      fields: []
    }

- delete.php
  ・フォーム削除API
  ・POST JSONで id を受け取る
  ・存在チェック → DELETE実行
  ・成功時は status: success を返す

---

【DB構造】

■ forms
- id TEXT PRIMARY KEY
- title TEXT
- description TEXT
- fields TEXT (JSON文字列)

■ answers
- id INTEGER PRIMARY KEY AUTOINCREMENT
- form_id TEXT
- field_id TEXT
- value TEXT
- created_at DATETIME DEFAULT CURRENT_TIMESTAMP

---

【現在の仕様】
- フォーム作成 → SQLiteに保存される
- フォーム一覧 → list.phpで取得
- フォーム詳細 → detail.phpで取得
- 削除 → delete.phpで削除

---

【既知の改善点 / 注意】
- JSONエラー時のフロント側「Unexpected token <」はPHPエラー出力が原因
- SQLiteファイル削除後でもテーブルは自動生成される設計
- save.phpとdb.phpで関数重複しないよう注意（redeclare対策）
- fieldsは必ずJSON encode/decodeで扱う
- delete.phpはrowCountチェックを使うが厳密必須ではない
